# CeFi（中心化金融）

中心化金融 (Centralized Finance, CeFi)，也稱為傳統金融，主要由銀行、券商、保險、第三方支付等**金融機構**作為中介提供服務。

特性：
- 金融機構**握有資產保管權**，可以凍結帳戶、限制提領、撤銷交易。
- 服務速度快、UI 友善、有客服與爭議處理機制。
- 必須通過機構的合規審查才能使用。

主流金融機構通常會要求三項合規：
- **KYC (Know Your Customer)**：機構必須驗證並掌握客戶真實身份（護照、地址、生物特徵等）。
- **AML (Anti-Money Laundering)**：反洗錢，要求機構偵測並通報可疑的資金流動，避免犯罪所得透過金融系統洗白。
- **CFT (Combating the Financing of Terrorism)**：反恐怖主義融資，與 AML 性質類似，特別針對恐怖活動相關的資金流。

# 智能合約平台的誕生

虛擬貨幣的歷史從 **比特幣 (Bitcoin, 2009)** 開始，但比特幣的設計目標主要是「點對點電子現金」，腳本語言不是圖靈完備，金融應用相對單純（轉帳、簡單時間鎖等）。

不久之後，**以太坊 (Ethereum, 2015)** 出現，主打**可程式化資產與金融**——這裡的「可程式化」指的就是**智能合約 (Smart Contract)**：可以把任意邏輯部署到鏈上，自動執行、無法竄改。從此鏈上應用從單純的轉帳擴展到借貸、交易所、衍生品、NFT 等。

# CeFi vs DeFi

DeFi (Decentralized Finance) 利用智能合約把金融服務放上鏈，**不需要中心化機構作為中介**。兩者主要差異：

| 維度 | CeFi | DeFi |
| --- | --- | --- |
| 是否許可 (Permissioned) | ✓ 須通過 KYC 才能使用 | ✗ 任何人持錢包即可使用 |
| 資產控制權 | 由機構代管 | 使用者自己持有私鑰 |
| 透明度 | 後台不公開 | 鏈上可全部查驗 |
| 中介角色 | 需要 | 由智能合約取代 |
| 速度 | 快（中央伺服器） | 較慢（受區塊時間限制） |
| 手續費 | 低 / 隱藏在價差 | Gas fee 公開 |
| 客服 / 救濟 | 有 | 幾乎沒有，操作錯誤難以挽回 |
| 監管 | 完整 | 法規不明確、灰色地帶 |

## DeFi 的優點
- **效率**：無中介、可組合性高（"money lego"），不同協定可互相串接。
- **透明**：所有交易與合約程式碼公開可查。
- **自主控管**：資產直接在自己的錢包，不會被第三方凍結。
- **無國界**：只要有網路與錢包就能參與，不受地理限制。

## DeFi 的代價
- 私鑰遺失即資產永久損失。
- 智能合約 bug / 攻擊（reentrancy、預言機操弄、閃電貸攻擊）造成的損失難以追回。
- 體驗門檻高，操作錯誤無客服可問。

# 為什麼需要 ERC？

ERC = **Ethereum Request for Comments**，是以太坊上的代幣與合約**介面標準**。在 ERC 出現前，不同代幣合約的函式名稱、參數順序、事件格式都可能不同，錢包與交易所必須為每個代幣特別客製化支援。

ERC 的目的：**統一介面，讓所有代幣行為一致**，錢包、交易所、DApp 才能不修改程式就支援新代幣。

常見標準：
- **ERC-20**：同質化代幣 (fungible token)，如 USDT、UNI、LINK。定義 `transfer`、`approve`、`balanceOf` 等基本介面。
- **ERC-721**：非同質化代幣 (NFT)，每個 token 都有獨立 `tokenId` 與唯一性。
- **ERC-1155**：多重代幣標準，一份合約可同時管理 fungible 與 non-fungible 資產，常見於遊戲道具。

只要合約遵循這些標準，就能無縫接入整個以太坊生態的工具鏈。
