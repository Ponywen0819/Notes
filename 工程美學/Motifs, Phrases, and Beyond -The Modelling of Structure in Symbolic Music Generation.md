 
### Intro
音樂本身擁有極複雜的結構，並彼此互相影響，其中包含高音、和聲、結奏、音色等等。

並且相對於故事分析，音樂的概念更加的複雜與抽象，此特性讓對音樂建模的任務變得困難。


此論文收集了 2007 年至 2023 年中針對音樂生成的論文，包括 RNN 、 GAN 與早期的深度學習模型。

並在最後聚焦在新興技術 ”子任務分解“ 的議題上。

### Methods
作者收集的方法中，可以分成 Symbolic Methods 與 Deep learning Methods
##### Symbolic Methods
主要應用方法為：馬可夫鏈、遺傳演算法等傳統學習方法。

| 方法             | 成果                         |
| -------------- | -------------------------- |
| 基於可變長度馬可夫的強化學習 | 模擬巴赫創意曲，能夠敏銳的辨識結構          |
| 馬可夫模型和SIACT模式  | 改進長期重複結構的獲取能力              |
| 馬可夫模型與可變鄰域搜尋   | 生成埃塞俄比亞傳統七弦琴Bagana風格的結構化音樂 |


雖然 Symbolic 方法擁有多種功能，但該方法嚴重受到特定模板的影響，在跨不同風格的音樂生成受到限制。

##### Deep Learning Methods
由於深度學習模型在學習特定模式的任務表現出良好的表現，因此後來的方法都以深度學習為主。
###### 早期
Google Magenta 在 2016 年推出了 Lookback RNN 與 Attention RNN，並且都針對原先的 RNN 進行改進

Lookback RNN 添加了自訂輸入與標籤，這項改動讓該模型更容易辨識前兩小節的內容與模式。

Attention RNN 則是融合了 Self Attention 機制與 LSTM ，增強模型對於長期資料辨識的能力。

在另一項研究中，使用一種基於擴張卷積的生成模型 —— WaveNet，來生成合乎給定旋律的音樂。
並發現 WaveNet 比起 LSTM 更能夠學習音樂的結構，


###### Transformer
2018 年，Google Brain 的研究員推出了 Music Transformer。
使用 Transformer 架構捕捉音樂的長期結構，並提出幾點改進
- 相對注意力
		與原始 transformer 依靠絕對位置編碼，本論文提出使用相對位置編碼作為代替。研究者認為對於音樂來說，音符之間的相對關係更為重要

但也是有幾點缺點
- 對於大於一分鐘的音樂片段，生成的內容會與初始提示開始分歧，整體結構較差。

2020 年，Queen Mary University of London 的研究員提出了 LSTM 與 Transformer 的變體，稱為 LSTUT。

![[Pasted image 20241211002951.png]]

該模型的特點在於使用 LSTM 取代原始的 position encoding，主要是要追求兩個優點
- position encoding 會讓模型需要學習編碼所提供的位置訊息，利用 LSTM 可以直接將輸入資料暫存與 context 中
- 透過輸入的 context ，transformer 可以學習到更高階層的建模

但他們的測試方法太過片面不完整，因此沒辦法客觀了解該模型的表現。

The Hong Kong Polytechnic University 的研究員認為音樂的重複邏輯重要度被低估，因此提出了針對音樂重複特徵的新模型 R-Transformer 。

該論文參考前人針對音樂結構所歸納出的規則，並聚焦於主題相關的種類

| 重複類型       | 音高規則        | 時值規則 | 適用場景         |
| ---------- | ----------- | ---- | ------------ |
| 嚴格重複 (StR) | 完全一致        | 完全一致 | 主題強調，增強識別性   |
| 轉調重複 (TrR) | 固定音程偏移      | 完全一致 | 調性轉換，和聲發展    |
| 後繼重複 (SuR) | 部分匹配 + 自由變化 | 自由變化 | 旋律變奏，熟悉感與創新性 |
| 同向重複 (HoR) | 運動方向一致 + 變化 | 自由變化 | 流動性與情緒表達     |
| 對稱重複 (SyR) | 對稱結構 + 變化   | 自由變化 | 對位音樂，結構性增強   |

### Subtask Decomposition
透過將音樂生成拆分成多個階段，以提升整體生成結果的連貫性，並兼顧局部細節。

在本論文所參考的論文中，都設計兩種不同目標的模型。

| Model                              | Contribution                                         |
| ---------------------------------- | ---------------------------------------------------- |
| **PopMNet (Wu, 2020)**             | 使用卷積生成對抗網路（GAN）生成旋律結構圖，並結合 RNN 生成旋律，展現了流行音樂的重複模式。    |
| Melons (Zou, 2022)                 | PopMNet 之改進，採用多邊結構圖表示複雜結構，並利用 Transformer 同時生成結構與旋律。 |
| **WuYun (Zhang, 2023)**            | 利用音樂理論萃取旋律骨架，採用 Transformer-XL 和編碼-解碼器生成旋律。          |
| **TeleMelody (Ju, 2021)**          | 通過模板生成旋律，結合音樂元素如調性與和弦進行，提升旋律與歌詞的結構匹配。                |
| **MeloForm (Lu, 2022)**            | 使用專家系統生成具有音樂形式的旋律，並用 Transformer 提升旋律豐富性。            |
| **Compose & Embellish (Wu, 2022)** | 基於前訓練策略生成鋼琴表演，使用結構分析增強旋律重複性與一致性。                     |

### Future Directions
作者認為音樂生成領域可以依據幾點大方向進行前進
1. 高級作曲技術
	作者認為目前所發表的模型針對基本的重複音樂結構已經有很好的表現，但在音樂的變化表現不佳，或許可以透過模擬高級作曲技巧可以改善模型在該方面的表現。
	
2. 高級抽象結構建模
	在子任務分解的章節，多種研究已經驗證兩段系統改善音樂生成的成效，因此作者認為讓機器學習更高階的抽象結構並將生成作業降解成多個步驟是一個重要的改進方向
